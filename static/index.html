LaneSwimmerTime (m:ss.hh)

The Colorado .do3 for Sport Systems is written only when you click Confirm & Export. Editing a time here collapses that lane to a single finish time.

Settings

Timing & Splits

One button per lane is standard. Adding a 2nd/3rd is optional — the extra timers' times are consolidated (median of 3, average of 2) per USA-Swimming rules. Change this, then Re-enroll to register the extra buttons.

Name display

How swimmer names show on the dashboard, spectator view and results (tick any combination — e.g. first name + surname initial for a public screen).

Sport Systems export (.do3)

The Colorado Dolphin file name must match what SPORTSYSTEMS looks for at Results Capture: {meet}-{event}-{heat}{round}{race}.do3. Event & heat come from the start list; set the meet number and round to match your meet.

Lane MAC Addresses

Button Registration

Re-assign lanes by the order you press the buttons: Lane 1 → … → Lane 6 → Starter.

Gateway (ESP32)

Fire the start signal (strobe + beep) to test the speaker, or reboot the gateway.

Starter Button

Wi-Fi (Bluetooth setup)

Send the gateway's Wi-Fi credentials to the ESP32 over Bluetooth. It saves them to flash and joins the network — no cable needed.

Roster

Import the seeded start list from Sport Systems — its heat‑sheet .txt export (or a Lenex .lef/.lxf).

Simulation Controls

Each tap on a lane = one wall touch, fired by the selected number of buttons. With splits on, tap again (after ~2 s) for the next length — the last length is the finish.
